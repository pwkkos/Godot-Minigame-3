## Devlog

I feel like the way I normally code is pretty similar to how it was encouraged in the lesson, except for the use of the print statements; I usually write a small amount of code at a time and test it before moving on, which helps a lot with finding mistakes quickly. I think my methodology is helping my process as it saves me the hassle of trying to fix errors within multiple different lines of code. Writing a smaller amount of lines makes it easier to debug the game because if there are any errors, I don't need to search the whole script to find what's wrong, I just need to check one section. I believe my process works really well for me and I'll most likely continue using it, however, I might try to follow the lesson's method of using print statements to test the code since I found it quite helpful.
